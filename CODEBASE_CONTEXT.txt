================================================================================
                              PROJECT OVERVIEW
================================================================================

Purplexity is an AI-powered conversational search engine that combines:
- Real-time web search via Tavily API
- LLM-generated answers via OpenRouter (with multiple model support)
- Multi-turn conversations with full context persistence
- User authentication via Supabase Auth (Google/GitHub OAuth)
- PostgreSQL database with Prisma ORM

CORE FLOW:
User Query → Tavily Web Search (10 results) → Context Engineering → 
OpenRouter LLM → Structured Response (Answer + Follow-ups + Sources) → 
Database Persistence

================================================================================
                           TECHNOLOGY STACK
================================================================================

RUNTIME & FRAMEWORK:
- Bun (v1.3.14) - JavaScript runtime (faster than Node.js)
- Express.js (v5.2.1) - REST API framework
- TypeScript (v5) - Type safety

AI & SEARCH:
- Tavily (@tavily/core v0.7.6) - Advanced web search with summaries
- OpenRouter (via OpenAI SDK v6.45.0) - Multi-provider LLM gateway
  - Supports: GPT-4, Claude, Gemini, DeepSeek, etc.
- Zod (v3) - Schema validation for LLM responses

DATABASE & AUTH:
- Supabase (@supabase/supabase-js v2.110.0) - BaaS (Auth + Storage)
- Prisma (@prisma/client v7.8.0) - Type-safe ORM
- PostgreSQL - Primary database

UTILITIES:
- dotenv (v17.4.2) - Environment variable management
- zod-to-json-schema (v3.25.2) - JSON schema generation

================================================================================
                        DATABASE SCHEMA (Prisma)
================================================================================

TABLES:

1. users
   - id (UUID, PK)
   - email (String, UNIQUE)
   - provider (AuthProvider: Github | Google)
   - name (String)
   - profileImage (String?)
   - createdAt, updatedAt

2. conversations
   - id (UUID, PK)
   - userId (UUID, FK → users.id, CASCADE DELETE)
   - title (String, default: "New Conversation")
   - createdAt, updatedAt
   - Indexes: userId, createdAt


3. messages
   - id (UUID, PK)
   - conversationId (UUID, FK → conversations.id, CASCADE DELETE)
   - role (MessageRole: user | assistant | system)
   - content (Text)
   - sources (JSON?, array of {title, url})
   - followUps (String[], array of follow-up questions)
   - createdAt
   - Indexes: conversationId, createdAt

4. queries (Legacy - standalone queries, not part of conversations)
   - id (UUID, PK)
   - userId (UUID, FK → users.id, CASCADE DELETE)
   - query (Text)
   - answer (Text)
   - followUps (String[])
   - sources (JSON)
   - createdAt
   - Indexes: userId, createdAt

RELATIONSHIPS:
User → Conversations (1:N)
User → Queries (1:N)
Conversation → Messages (1:N)

CASCADE DELETES:
- Delete user → deletes all conversations & queries
- Delete conversation → deletes all messages

"You are Purplexity, an expert assistant. Answer queries based on web search

Generate 3-5 relevant follow-up questions."

PROMPT_TEMPLATE (prompt.ts):
## Web search results
{{WEB_SEARCH_RESULTS}}

## USER_QUERY
{{USER_QUERY}}

FLOW:
1. Replace {{WEB_SEARCH_RESULTS}} with JSON.stringify(webSearchResults)
2. Replace {{USER_QUERY}} with actual user query
3. Send to LLM with SYSTEM_PROMPT
4. Parse JSON response with Zod schema

================================================================================
                        TYPICAL REQUEST FLOW
================================================================================

EXAMPLE: User sends "What is Rust?" in an existing conversation

1. Frontend → POST /conversations/{id}/messages
   Body: { message: "What is Rust?" }

2. Server validates conversationId exists

3. webSearch.service.searchWeb("What is Rust?")
   - Calls Tavily API
   - Returns 10 web results with content summaries

4. ai.service.generateResponse(query, results)
   - Builds context: SYSTEM_PROMPT + PROMPT_TEMPLATE + web results
   - Calls OpenRouter with model: "openrouter/auto"
   - Receives JSON: { answer: "...", followUps: [...] }

5. conversation.service.addMessageToConversation()
   - Creates user message in DB
   - Creates assistant message in DB (with sources & followUps)
   - Updates conversation.updatedAt

6. If first message: generateConversationTitle()
   - Takes first 50 chars of user message as title

7. Server responds with:
   {
     userMessage: { id, role: "user", content, ... },
     assistantMessage: { id, role: "assistant", content, sources, followUps },
     sources: [{ title, url }],
     followUps: ["question 1", "question 2", ...]
   }

================================================================================
                        CONTRIBUTION GUIDELINES
================================================================================

HOW TO CONTRIBUTE:

1. Fork the repository
2. Create a feature branch (feature/my-new-feature)
3. Make changes following code style
4. Add tests for new features
5. Update documentation
6. Submit pull request with clear description

CODE STYLE:
- Use TypeScript strict mode
- Follow existing naming conventions
- Add JSDoc comments for public APIs
- Keep functions small and focused
- Use async/await (not callbacks)
- Handle errors gracefully

TESTING REQUIREMENTS:
- Unit tests for service functions
- Integration tests for API endpoints
- E2E tests for critical user flows
- Test coverage > 80%

DOCUMENTATION REQUIREMENTS:
- Update API_DOCUMENTATION.md for new endpoints
- Update DATABASE_SCHEMA.md for schema changes
- Add inline comments for complex logic
- Update CHANGELOG.md

================================================================================
                        GLOSSARY
================================================================================

TERMS:

Conversation: A multi-turn chat session between user and AI
Message: Single user question or AI response within conversation
Query: Legacy standalone search (not part of conversation)
Source: Web page citation with title, URL, and snippet
Follow-up: AI-generated suggested next question
Context Engineering: Building effective prompts for LLM
RAG: Retrieval-Augmented Generation (web search + LLM)
SSE: Server-Sent Events (real-time streaming)
RLS: Row Level Security (Supabase database permissions)
JWT: JSON Web Token (authentication)
UUID: Universally Unique Identifier (non-sequential ID)
Prisma: Type-safe ORM for database queries
Supabase: Backend-as-a-Service (auth, database, storage)
OpenRouter: LLM provider aggregator (access multiple models)
Tavily: AI-optimized web search API

ACRONYMS:

API: Application Programming Interface
REST: Representational State Transfer
CRUD: Create, Read, Update, Delete
ORM: Object-Relational Mapping
JWT: JSON Web Token
SSE: Server-Sent Events
RLS: Row Level Security
UUID: Universally Unique Identifier
LLM: Large Language Model
RAG: Retrieval-Augmented Generation
TTL: Time To Live (cache expiration)
CORS: Cross-Origin Resource Sharing
SQL: Structured Query Language
JSON: JavaScript Object Notation
HTTP: Hypertext Transfer Protocol
HTTPS: HTTP Secure
CDN: Content Delivery Network
MAU: Monthly Active Users
